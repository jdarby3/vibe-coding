The user has a raw idea. Help them shape it before committing to a PRD.

1. **Listen**: Let them dump their thoughts. Don't interrupt with structure yet.
2. **Mirror**: Restate the idea in one sentence. Ask "Is that right?"
3. **Expand**: Ask 3 targeted questions:
   - Who has this problem today, and what do they do about it?
   - What's the smallest thing you could build to test this?
   - What would make you abandon this idea?
4. **Sketch**: Write a 5-line elevator pitch: problem, who, solution, differentiator, next step.
5. **Fork**: Offer two paths — "Want me to write a PRD?" or "Want me to grill this first?"
