# /project:blueprint

Write a structured plan for the proposed change. Do not begin implementation until the developer explicitly says to proceed.

## Plan Format

**What**
One sentence describing the change.

**Why**
The motivation or problem being solved.

**Approach**
Step-by-step breakdown of how this will be implemented.

**Files Affected**
List every file that will be created, modified, or deleted.

**New Dependencies**
Any npm packages that will be added. If none, state "None".

**Risks**
Anything that could go wrong or has uncertain scope.

---

After writing the plan, stop. Do not write any code. Wait for the developer to respond with approval or changes before proceeding.
