Read `.claude/state/prd.md`. Break it into an ordered implementation plan.

For each task:
- **Task**: one sentence description.
- **Files**: which files will be created or changed.
- **Depends on**: which prior tasks must be done first.
- **Estimate**: S / M / L (hours: <1 / 1-4 / 4+).
- **Acceptance**: how do we know it's done?

Write the plan to `.claude/state/tasks.md` as a checklist:
```
- [ ] Task 1 (S) — description
- [ ] Task 2 (M) — description, depends on Task 1
```

Order tasks so the riskiest/hardest come first. Front-load learning.
